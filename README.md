# Craigslist Dataset

Репозиторий содержит сквозной конвейер машинного обучения для предсказания стоимости автомобилей на основе их технических характеристик и текстового описания. Проект полностью оптимизирован и подготовлен к деплою в продакшен.

---

## Архитектура и этапы обработки данных

Проект разделен на изолированные этапы предобработки, что исключает утечку данных (data leakage) на этапе валидации и обеспечивает стабильную работу модели при инференсе:

1. **Очистка и фильтрация данных:** Удаление аномалий, обработка пропусков и фильтрация выбросов в целевой переменной и числовых признаках.
2. **Feature Engineering (Проектирование признаков):**
   * **One-Hot Encoding:** Маломощные категориальные признаки развернуты в отдельные бинарные столбцы.
   * **Target Encoding:** Высокомощные категориальные признаки (`model`, `region`, `manufacturer`) закодированы сглаженными средними значениями целевой переменной.
   * **Текстовый векторный анализ (TF-IDF):** Сырой текст из колонки `description` очищен от стоп-слов и преобразован в 50 наиболее информативных числовых признаков с помощью `TfidfVectorizer`.
3. **Feature Selection (Отбор признаков):** На основе анализа **SHAP (TreeExplainer)** были определены и удалены малоинформативные признаки, не оказывающие значимого влияния на предсказание модели.

---

## Результаты оптимизации модели

В качестве базового алгоритма используется ансамбль **RandomForestRegressor**. В ходе оптимизации конвейера были достигнуты следующие показатели:

* **Оптимизация гиперпараметров:** С помощью поиска `GridSearchCV` и кросс-валидации метрика качества на валидационной выборке улучшилась на **2%**. Ошибка по RMSE зафиксирована в районе **18%**, а коэффициент детерминации ($R^2$) составил **0.9364**.
* **Фильтрация признаков по SHAP:** Удаление слабых фич позволило сократить размерность признакового пространства **почти на 10%**. Модель стала более компактной и быстрой, существенно снизился риск переобучения, а качество предсказания дополнительно выросло на **0.22%**.

## Production Assets

После обучения конвейер экспортирует четыре компактных бинарных файла:

* `model.joblib` — оптимизированная модель Случайного Леса.
* `encoder.joblib` — обученный TargetEncoder для категориальных фич.
* `tfidf_vectorizer.joblib` — обученный векторизатор для текстовых описаний.
* `important_features.joblib` — сериализованный список финальных признаков, одобренных SHAP и готовых для выравнивания через `.reindex()`.
